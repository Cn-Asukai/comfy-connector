# comfy-connector 单元测试计划

## 决策摘要

| 决策项 | 方案 |
|--------|------|
| 测试框架 | Go 标准 `testing` + `testify` (assert/require) |
| HTTP 隔离 | `net/http/httptest` fake server |
| WebSocket 隔离 | `gorilla/websocket` + `net/http/httptest` fake WS server |
| 测试组织 | `_test` 后缀包（包外测试，只测公开 API） |
| Mock 策略 | `testify/mock`（用于 scheduler 测试中的 queue 和 handler mock） |
| 优先级 | queue/memory → scheduler → comfyui |

---

## 前置步骤：添加依赖

```bash
go get github.com/stretchr/testify
```

## 1. queue/memory 测试 (`queue/memory/queue_test.go`)

**包名：** `memory_test`

### 1.1 Enqueue

| 场景 | 验证点 |
|------|--------|
| 正常入队 | job 入队后 Size()==1，无错误 |
| 重复 ID | 返回 `queue.ErrJobDuplicate` |
| context 已取消 | 返回 `ctx.Err()`，但 Enqueue 内部实际不检查 ctx（保持现状，验证无 panic） |

### 1.2 Dequeue

| 场景 | 验证点 |
|------|--------|
| 空队列 timeout=-1 | 立即返回 (nil, nil) |
| 空队列 timeout>0 | 等待超时后返回 (nil, nil) |
| 空队列 context 取消 | 返回 (nil, ctx.Err()) |
| 有 pending job | 返回 job 且该 job 不在 pendingIDs 中，Size() 减 1 |
| 优先级排序 | 高优先级 job 先出队（入队 priority=10, 1，验证先出 priority=10） |
| aging 效果 | 先入队低优先级 job，等待一段时间后入队高优先级 job，验证低优先级因 aging 先出队 |

### 1.3 Ack / Nack

| 场景 | 验证点 |
|------|--------|
| 存在的 running job Ack | 返回 nil |
| 不存在的 job Ack | 返回 `queue.ErrJobNotFound` |
| 非 running 状态的 job Ack | 返回 `queue.ErrJobNotRunning` |
| Nack 同上 | 同上 |

### 1.4 Get

| 场景 | 验证点 |
|------|--------|
| 存在的 job | 返回 job 副本，修改返回的 job 不影响队列内的原始 job |
| 不存在的 job | 返回 `queue.ErrJobNotFound` |

### 1.5 Cancel

| 场景 | 验证点 |
|------|--------|
| 存在的 pending job | 返回 nil，job 从 pendingIDs 移除，Size() 减 1 |
| 不存在的 job | 返回 `queue.ErrJobNotFound` |

### 1.6 Size

| 场景 | 验证点 |
|------|--------|
| 空队列 | 返回 0 |
| 有 n 个 pending job | 返回 n |

### 1.7 并发安全

| 场景 | 验证点 |
|------|--------|
| 10 goroutine 并发 Enqueue + 10 goroutine 并发 Dequeue | 所有 job 被正确分发，无 race condition（`go test -race`） |

| 场景 | 验证点 |

## 风险与注意事项

1. **Ack/Nack 不修改 job 状态**：当前 `memory.Queue` 的 `Ack` 和 `Nack` 只做校验，不修改 job 状态（scheduler 直接修改 job 指针）。这违反了 Queue 抽象应有的一致性，但测试时只需要验证接口约定。
2. **WebSocket 测试复杂度**：`gorilla/websocket` 与 `httptest` 的集成需要使用 `http.Hijacker` 接口进行协议升级，需要仔细处理。
